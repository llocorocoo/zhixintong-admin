# 职信通后台管理系统 - 项目设计文档

## 1. 项目概述

职信通后台管理系统是一个面向平台运营方和渠道商的 PC 端 Web 管理后台，用于管理渠道商、订单、交易等核心业务数据。系统支持多角色登录，按角色控制菜单可见性和数据范围。

## 2. 技术方案

| 项目       | 选型                          |
| ---------- | ----------------------------- |
| 框架       | React 18 + TypeScript         |
| 构建工具   | Vite                          |
| UI 组件库  | Ant Design 5                  |
| 路由       | React Router v6               |
| 状态管理   | Zustand                       |
| 请求       | Axios                         |
| 数据模拟   | Mock.js（原型阶段）           |
| 代码规范   | ESLint + Prettier             |
| 部署       | GitHub → Vercel（免费额度）   |

## 3. 角色与权限

| 角色       | 登录方式           | 可见菜单                                       | 数据范围     |
| ---------- | ------------------ | ---------------------------------------------- | ------------ |
| 平台管理员 | 后台账号密码登录   | 全部菜单                                       | 全平台数据   |
| 渠道商     | 管理员分配账号登录 | 渠道商管理（仅查看自己）、订单管理、交易明细   | 仅本渠道数据 |

## 4. 功能模块

### 4.1 渠道商管理

| 序号 | 功能项           | 功能说明                                                           | 可见角色 |
| ---- | ---------------- | ------------------------------------------------------------------ | -------- |
| 1    | 渠道商列表       | 查看所有渠道商，支持新增、编辑、启用/停用                          | 管理员   |
| 2    | 渠道商基本信息   | 名称、联系人、联系方式、渠道类型（纯渠道/OEM）等                   | 管理员   |
| 3    | 生成推广链接     | 为渠道商生成带渠道码的专属推广链接                                 | 管理员   |
| 4    | 查看推广链接/二维码 | 渠道商查看自己的专属推广链接和二维码                             | 渠道商   |
| 5    | OEM 域名绑定     | 渠道商类型为 OEM 时，支持绑定渠道商自有域名，系统做域名映射        | 管理员   |
| 6    | OEM Logo 定制    | 渠道商类型为 OEM 时，支持上传自定义 Logo，用于报告页面等场景展示   | 管理员   |

### 4.2 账号管理

| 序号 | 功能项       | 功能说明                       | 可见角色 |
| ---- | ------------ | ------------------------------ | -------- |
| 7    | 渠道商账号开设 | 管理员为渠道商创建登录账号   | 管理员   |
| 8    | 账号启用/停用  | 控制渠道商账号的可用状态     | 管理员   |
| 9    | 密码重置       | 管理员可重置渠道商账号密码   | 管理员   |

### 4.3 订单管理

| 序号 | 功能项   | 功能说明                                                                         | 可见角色       |
| ---- | -------- | -------------------------------------------------------------------------------- | -------------- |
| 10   | 订单列表 | 管理员查看全部订单；渠道商查看本渠道订单。支持按渠道、时间等条件筛选              | 管理员/渠道商  |
| 11   | 订单详情 | 用户信息、下单时间、报告类型、金额、渠道来源等                                    | 管理员/渠道商  |

### 4.4 交易明细

| 序号 | 功能项       | 功能说明                                                                                 | 可见角色       |
| ---- | ------------ | ---------------------------------------------------------------------------------------- | -------------- |
| 12   | 交易流水记录 | 管理员查看全平台交易流水；渠道商查看本渠道交易明细。支持按时间、渠道、金额筛选            | 管理员/渠道商  |
| 13   | 数据导出     | 支持将交易数据导出为 Excel 等格式                                                        | 管理员         |

> 备注：数据导出为后续扩展功能。

### 4.5 系统配置

| 序号 | 功能项       | 功能说明                     | 可见角色 |
| ---- | ------------ | ---------------------------- | -------- |
| 14   | 报告相关配置 | 报告模板、内容等相关配置项   | 管理员   |
| 15   | 其他常规配置 | 预留配置模块，具体内容待定   | 管理员   |

> 备注：系统配置模块为后续扩展功能。

## 5. 页面结构

```
登录页
└── 主布局（侧边栏 + 顶部栏 + 内容区）
    ├── 首页/仪表盘
    ├── 渠道商管理
    │   ├── 渠道商列表页
    │   ├── 新增/编辑渠道商（抽屉或弹窗）
    │   └── 渠道商详情页（推广链接、OEM 配置等）
    ├── 账号管理
    │   └── 账号列表页（新增、启用/停用、重置密码）
    ├── 订单管理
    │   ├── 订单列表页
    │   └── 订单详情页（弹窗或新页面）
    ├── 交易明细
    │   └── 交易流水列表页
    └── 系统配置（后续扩展）
        ├── 报告配置
        └── 其他配置
```

## 6. 路由设计

| 路径                       | 页面             | 权限       |
| -------------------------- | ---------------- | ---------- |
| `/login`                   | 登录页           | 公开       |
| `/`                        | 仪表盘           | 登录后     |
| `/channel`                 | 渠道商列表       | 管理员     |
| `/channel/:id`             | 渠道商详情       | 管理员     |
| `/channel/my`              | 我的渠道信息     | 渠道商     |
| `/account`                 | 账号管理         | 管理员     |
| `/order`                   | 订单列表         | 全部角色   |
| `/order/:id`               | 订单详情         | 全部角色   |
| `/transaction`             | 交易明细         | 全部角色   |
| `/settings`                | 系统配置         | 管理员     |

## 7. 部署方案

```
本地开发 → Git Push → GitHub 仓库 → Vercel 自动构建部署
```

- **GitHub 仓库**: 新建 `zhixintong-admin` 仓库
- **Vercel 项目**: 关联 GitHub 仓库，自动检测 Vite 项目
- **构建配置**:
  - Build Command: `npm run build`
  - Output Directory: `dist`
  - Install Command: `npm install`
- **路由重写**: SPA 单页应用，所有路径回退到 `index.html`

Vercel 免费额度包含：
- 每月 100GB 带宽
- 每次构建 45 秒超时（Vite 构建通常 < 10 秒）
- 自动 HTTPS
- 自定义域名支持
