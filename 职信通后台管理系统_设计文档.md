# 职信通后台管理系统 - 项目设计文档

## 1. 项目概述

职信通后台管理系统是一个面向平台运营方和渠道商的 PC 端 Web 管理后台，用于管理渠道商、订单、交易等核心业务数据。系统支持多角色登录（管理员 / 渠道商），按角色控制菜单可见性和数据范围。当前处于原型阶段，使用 Mock 数据。

## 2. 技术方案

| 项目       | 选型                          |
| ---------- | ----------------------------- |
| 框架       | React 18 + TypeScript         |
| 构建工具   | Vite 8                        |
| UI 组件库  | Ant Design 5                  |
| 路由       | React Router v6（BrowserRouter）|
| 状态管理   | Zustand（useAuth、useChannels）|
| 请求       | Axios（已引入，待对接）       |
| 数据模拟   | 本地 Mock 数据（`src/mock/data`）|
| 代码规范   | ESLint + Prettier             |
| 部署       | GitHub → Vercel（免费额度）   |

## 3. 角色与权限

| 角色       | 登录方式           | 可见菜单                                                         | 数据范围     |
| ---------- | ------------------ | ---------------------------------------------------------------- | ------------ |
| 平台管理员 | 后台账号密码登录（admin / admin123） | 仪表盘、渠道商管理、账号管理、订单管理、交易明细、系统配置、个人中心 | 全平台数据   |
| 渠道商（纯渠道） | 管理员分配账号登录（channel01 / 123456） | 仪表盘、我的渠道、订单管理、交易明细、个人中心         | 仅本渠道数据 |
| 渠道商（OEM）    | 管理员分配账号登录（channel02 / 123456） | 仪表盘、我的渠道、订单管理、交易明细、个人中心         | 仅本渠道数据 |

权限守卫通过 `AuthGuard` 组件实现，支持 `roles` 属性控制页面访问。

## 5. 页面结构

```
登录页（/login）
└── 主布局（侧边栏 + 顶部栏 + 内容区）
    ├── 仪表盘（/）— 统计卡片
    ├── 渠道商管理 [管理员]
    │   ├── 渠道商列表页（/channel）— 搜索筛选 + 表格 + 新增/编辑弹窗
    │   └── 渠道商详情页（/channel/:id）— 基本信息 + 推广链接/OEM配置
    ├── 我的渠道 [渠道商]
    │   └── 渠道信息页（/channel/my）— 渠道信息 + 推广链接/二维码
    ├── 账号管理 [管理员]
    │   └── 账号列表页（/account）— 表格 + 新增弹窗 + 启用/停用/重置密码
    ├── 订单管理
    │   └── 订单列表页（/order）— 搜索筛选 + 表格 + 详情弹窗
    ├── 交易明细
    │   └── 交易流水列表页（/transaction）— 搜索筛选 + 表格
    ├── 个人中心
    │   └── 个人资料页（/user-center/profile）— 头像信息 + 密码/手机/邮箱管理
    └── 系统配置 [管理员]
        └── 配置页面（/settings）— 占位（待开发）
```

## 6. 路由设计

| 路径                       | 页面             | 权限       | 组件文件                          |
| -------------------------- | ---------------- | ---------- | --------------------------------- |
| `/login`                   | 登录页           | 公开       | `pages/Login/index.tsx`           |
| `/`                        | 仪表盘           | 登录后     | `pages/Dashboard/index.tsx`       |
| `/channel`                 | 渠道商列表       | 管理员     | `pages/Channel/index.tsx`         |
| `/channel/:id`             | 渠道商详情       | 管理员     | `pages/Channel/Detail.tsx`        |
| `/channel/my`              | 我的渠道信息     | 渠道商     | `pages/Channel/MyChannel.tsx`     |
| `/account`                 | 账号管理         | 管理员     | `pages/Account/index.tsx`         |
| `/order`                   | 订单列表         | 全部角色   | `pages/Order/index.tsx`           |
| `/transaction`             | 交易明细         | 全部角色   | `pages/Transaction/index.tsx`     |
| `/settings`                | 系统配置         | 管理员     | `pages/Settings/index.tsx`        |

| `/user-center/profile`     | 个人资料         | 登录后     | `pages/UserCenter/Profile.tsx`    |

## 8. 部署方案

```
本地开发 → Git Push → GitHub 仓库 → Vercel 自动构建部署
```

- **GitHub 仓库**: `zhixintong-admin`
- **Vercel 项目**: 关联 GitHub 仓库，自动检测 Vite 项目
- **构建配置**:
  - Build Command: `npm run build`
  - Output Directory: `dist`
  - Install Command: `npm install`
- **路由重写**: SPA 单页应用，所有路径回退到 `index.html`
